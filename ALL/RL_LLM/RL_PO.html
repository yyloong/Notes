公式校注

  • PPO [已修正①]：GAE 指数 $(\lambda\gamma)^{l+t}$ → $(\lambda\gamma)^{l}$，求和哑变量为 $l$，$t$ 为当前时间步。
  • PPO [已修正②]：TD 误差中 $V(y_{i,\leq l+t})$ → $V(y_{i,\leq l+1})$（下一个 token 位置）。
  • FIPO [已修正]：$\mathbb{I}(\cdot)$ 内条件改为 IS 比值 $\leq c$（截断阈值）；$\hat{A}_{i,t}$ 乘以 $\mathrm{clip}(\exp(\text{FKL}_{i,t}),\cdot)$ 作为 future-aware 加权因子。
  • 渲染修正：所有绝对值 $\lvert y_i\rvert$ 改用 \lvert...\rvert（避免裸 | 被 Markdown 解析为列分隔符）；条件概率中的 $y_{i,{<}t}$ 用 {<} 包裹防止被识别为 HTML 标签。
  • GRPO/DAPO [说明]：奖励统一为序列级 $r_i = R(x,y_i)$，与原论文一致。
  • GSPO [说明]：IS ratio 取序列联合概率比的 $\frac{1}{\lvert y_i\rvert}$ 次方，实现长度归一化。
  • GiGPO [说明]：双级优势：$A^E_i$ 为 episode 级 group-relative 优势；$A^S_{i,t}$ 的比较集合 $G^S(s_{i,t})$ 收集所有轨迹 $j$、所有时间步 $t'$ 中到达同一状态 $s_{i,t}$ 的 (action, reward) 对，而非只比较同一 $t$ 时刻——这是 GiGPO 的核心设计。注意：GiGPO 中的 $t$ 是 序列步（reasoning step/turn）级别，与其他算法中 token 级别的 $t$ 含义不同。
